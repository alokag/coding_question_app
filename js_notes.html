JavaScript Interview Notes – 1‑Day Revision Guide

1. Variables: let, const, var

JavaScript provides three ways to declare variables:

  • var – function‑scoped, hoisted, outdated.
  • let – block‑scoped, used for reassignable values.
  • const – block‑scoped, cannot be reassigned.
let count = 1;
count++;

const user = { name: "Monu" };
user.name = "Alok"; // allowed – object mutation

Real‑life usage: const for configs, let for loops and temp variables.

2. Hoisting

JavaScript moves declarations to the top of their scope. var is hoisted with undefined, while let and const exist in TDZ (Temporal Dead Zone).

console.log(a); // undefined
var a = 10;

console.log(b); // ReferenceError
let b = 20;

3. Functions: Declaration vs Expression vs Arrow

// Declaration
function add(a, b) { return a + b; }

// Expression
const mul = function(a, b) { return a * b; }

// Arrow function
const sub = (a, b) => a - b;

Usage: Arrow functions are best for short callbacks and array operations.

4. Execution Context & Call Stack

JavaScript runs code inside execution contexts:

  • Global Execution Context
  • Function Execution Context

Call stack manages function invocation order.

5. Closures

A closure is created when a function remembers its outer scope even after the outer function has executed.

function outer() {
  let counter = 0;
  return function() {
    counter++;
    return counter;
  };
}
const inc = outer();
inc(); // 1
inc(); // 2

Use case: Private variables, memoization, custom hooks.

6. Event Loop, Microtasks & Macrotasks

Handles async operations and keeps JS non‑blocking.

console.log("A");
setTimeout(() => console.log("B"));
Promise.resolve().then(() => console.log("C"));
console.log("D");
// Output: A D C B

7. Async Programming: Promises & async/await

async function load() {
  const data = await fetch('/api');
  return data.json();
}

Real‑life: API calls, file operations, database operations.

8. this Binding

const obj = {
  name: "JS",
  say() { console.log(this.name); }
};
obj.say();

Note: Arrow functions don't have their own this.

9. Prototypes & Inheritance

function Person(name){ this.name = name; }
Person.prototype.say = function(){ console.log(this.name); };

Real‑life: Reusable object methods.

10. Classes

class Car {
  constructor(model) { this.model = model; }
  start(){ console.log("Starting", this.model); }
}

11. Arrays & High‑Order Methods

map, filter, reduce are essential.

const prices = [10,20,30];
const total = prices.reduce((a,b) => a + b, 0);

12. Spread & Rest

const arr = [1,2,3];
const newArr = [...arr, 4];

function sum(...nums) { return nums.reduce((a,b) => a+b); }

13. Destructuring

const user = {name: "Alok", age: 25};
const {name, age} = user;

14. Modules (import/export)

// utils.js
export function add(a,b){ return a+b; }

// main.js
import { add } from './utils.js';

15. DOM Manipulation

document.querySelector("#box").innerText = "Updated";

16. Event Delegation

document.body.addEventListener("click", e => {
  if(e.target.matches("button")){
    console.log("Button clicked");
  }
});

17. Debounce & Throttle

Used in scroll, resize, input events.

function debounce(fn, delay){
  let t;
  return (...args) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...args), delay);
  };
}

18. Error Handling

try {
  risky();
} catch (err) {
  console.error(err);
}

fetch('/api')
 .then(res => res.json())
 .then(console.log)
